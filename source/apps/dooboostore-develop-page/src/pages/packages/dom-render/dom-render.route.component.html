dom-render

HTML view template engine with DOM control and reactive rendering

DOM 제어와 반응형 렌더링을 지원하는 HTML 뷰 템플릿 엔진

Installation

Install the package and its dependencies:

패키지와 의존성을 설치하세요:

npm 
npm install @dooboostore/dom-render reflect-metadata
pnpm 
pnpm add @dooboostore/dom-render reflect-metadata

Features

Template Engine

Powerful HTML template engine with expression binding and control structures

표현식 바인딩과 제어 구조를 지원하는 강력한 HTML 템플릿 엔진

DOM Control

Direct DOM manipulation and reordering with efficient rendering

효율적인 렌더링을 통한 직접적인 DOM 조작 및 재정렬

Proxy-based Reactivity

All internal variables managed by DomRenderProxy for automatic updates

자동 업데이트를 위해 DomRenderProxy로 관리되는 모든 내부 변수

Component System

ComponentBase with decorators for attributes, queries, and events

속성, 쿼리, 이벤트를 위한 데코레이터를 지원하는 ComponentBase

Form Handling

Built-in form binding with validation support

검증 지원을 포함한 내장 폼 바인딩

Routing System

Declarative routing with path matching and parameters

경로 매칭과 매개변수를 지원하는 선언적 라우팅

Quick Start

Basic Usage

Get started with a simple template binding example:

간단한 템플릿 바인딩 예제로 시작하세요:

TypeScript 
import { DomRender } from '@dooboostore/dom-render';

class AppData {
  name: string = 'my name is dom-render';
  color: string = '#ff0000';
  
  changeData() {
    this.name = 'Updated name: ' + new Date().toLocaleTimeString();
    this.color = '#' + Math.floor(Math.random()*16777215).toString(16);
  }
}

const data = new AppData();
DomRender.run(data, document.querySelector('#app')!);
HTML 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>DOM Render Example</title>
</head>
<body id="app">
  <h1 style="color: \${this.color}\$">\${this.name}\$</h1>
  <button dr-event-click="this.changeData()">Change Data</button>
</body>
</html>

Template Expressions

Use different expression types for text and HTML content:

텍스트와 HTML 콘텐츠에 다양한 표현식 타입을 사용하세요:



                        <!-- HTML content (unescaped) -->
<div># {this.htmlContent}#</div>


                        <!-- Text content (escaped) -->
<div>\${this.name}\$</div>

<!-- Attribute binding -->
<input type="text" value="\${this.inputValue}\$" style="\${'color: ' + this.textColor}\$">

Control Structures

Use directives for conditional rendering and loops:

조건부 렌더링과 반복을 위한 디렉티브를 사용하세요:

<!-- Conditional rendering -->
<div dr-if="this.isVisible">This is visible</div>
<div dr-if="this.user.role === 'admin'">Admin content</div>

<!-- Loop rendering -->
<ul>
  <li dr-for-of="this.items">\${#it#.name}\$ - \${#it#.price}\$</li>
</ul>

<!-- Traditional for loop -->
<div dr-for="var i = 0; i &lt; this.count; i++">
  Item \${i + 1}\$
</div>

Template Directives

DOM Render provides a comprehensive set of directives for template control:

DOM Render는 템플릿 제어를 위한 포괄적인 디렉티브 세트를 제공합니다:

dr-if

Conditional element rendering

조건부 요소 렌더링

dr-for / dr-for-of

Loop rendering with iteration

반복을 통한 루프 렌더링

dr-event-*

Event binding (click, input, change, etc.)

이벤트 바인딩 (클릭, 입력, 변경 등)

dr-value-link

Two-way data binding for form inputs

폼 입력을 위한 양방향 데이터 바인딩

dr-class

Dynamic CSS class binding

동적 CSS 클래스 바인딩

dr-style

Dynamic inline style binding

동적 인라인 스타일 바인딩

dr-attr

Dynamic attribute binding

동적 속성 바인딩

dr-this

Context switching for nested objects

중첩 객체를 위한 컨텍스트 전환

dr-form

Form data binding with validation

검증을 포함한 폼 데이터 바인딩

dr-checked / dr-disabled

Boolean attribute binding

불린 속성 바인딩

dr-appender

Dynamic list management

동적 리스트 관리

dr-route-match

Route-based conditional rendering

라우트 기반 조건부 렌더링

Advanced Features

Component System

Create reusable components with ComponentBase and decorators:

ComponentBase와 데코레이터로 재사용 가능한 컴포넌트를 생성하세요:

import { ComponentBase, attribute, query, event } from '@dooboostore/dom-render';

export class UserCard extends ComponentBase<{ 'user-name': string, 'user-email': string }> {
    @attribute
    userName: string = 'Default User';

    @attribute({ name: 'user-email' })
    userEmail: string = '';

    @query('button.edit')
    private editButton: HTMLButtonElement;

    @query('span.status')
    private statusElements: HTMLSpanElement[];

    @event({ query: 'button.edit', name: 'click' })
    onEditClick(e: MouseEvent) {
        console.log('Edit clicked for:', this.userName);
    }

    onInitRender() {
        if (this.editButton) {
            this.editButton.style.display = this.userEmail ? 'block' : 'none';
        }
    }
}
Usage in HTML 
<user-card user-name="John Doe" user-email="john@example.com">
  <div>
    <h3>\${@this@.userName}\$</h3>
    <p>\${@this@.userEmail}\$</p>
    <span class="status">Active</span>
    <button class="edit">Edit User</button>
  </div>
</user-card>

Form Handling & Validation

Handle forms with built-in validation support:

내장 검증 지원으로 폼을 처리하세요:

import { DomRender } from '@dooboostore/dom-render';
import { FormValidator, NotEmptyValidator, RegExpTestValidator } from '@dooboostore/dom-render/validator';

class UserForm {
    form: FormValidator = new FormValidator();

    constructor() {
        this.form.name = new NotEmptyValidator();
        this.form.email = new RegExpTestValidator(/^[^\s@]+@[^\s@]+\.[^\s@]+$/);
        this.form.age = new NotEmptyValidator();
    }

    submit() {
        if (this.form.valid()) {
            console.log('Form is valid:', this.form);
            // Process form data
        } else {
            console.log('Form has errors');
            // Show validation errors
        }
    }
}
Form HTML 
<form dr-form="@this@.form" dr-event-submit="@this@.submit(); $event.preventDefault();">
  <div>
    <label>Name:</label>
    <input name="name" type="text" required>
  </div>
  <div>
    <label>Email:</label>
    <input name="email" type="email" required>
  </div>
  <div>
    <label>Age:</label>
    <input name="age" type="number" required>
  </div>
  <button type="submit">Submit</button>
</form>

Routing System

Implement client-side routing with declarative components:

선언적 컴포넌트로 클라이언트 사이드 라우팅을 구현하세요:

import { DomRender, DomRenderRunConfig } from '@dooboostore/dom-render';

class App {
    currentPath: string = '/';
    
    constructor() {
        // Router will automatically update currentPath
    }
}

const config: DomRenderRunConfig<App> = {
    window: window,
    routerType: 'path' // or 'hash'
};

const domRender = new DomRender({
    rootObject: new App(),
    target: document.querySelector('#app')!,
    config: config
}, { autoGo: '/' });
Routing HTML 
<nav>
  <button dr-event-click="$router?.go({path:'/'})">Home</button>
  <button dr-event-click="$router?.go({path:'/about'})">About</button>
  <button dr-event-click="$router?.go({path:'/user/123'})">User 123</button>
</nav>

<main>
  <dr-route-match value="/">
    <h1>Home Page</h1>
  </dr-route-match>

  <dr-route-match value="/about">
    <h1>About Page</h1>
  </dr-route-match>

  <dr-route-match value="/user/{id:[0-9]+}">
    <h1>User Profile</h1>
    <p>User ID: \${$router.pathData?.id}\$</p>
  </dr-route-match>
</main>

Core Concepts

DomRender

Main class for initializing and managing template rendering

템플릿 렌더링을 초기화하고 관리하는 메인 클래스

DomRenderProxy

Proxy system for automatic change detection and DOM updates

자동 변경 감지와 DOM 업데이트를 위한 프록시 시스템

ComponentBase

Base class for creating reusable components with lifecycle hooks

생명주기 훅을 포함한 재사용 가능한 컴포넌트 생성을 위한 기본 클래스

Template Directives

Special attributes (dr-*) for controlling DOM behavior

DOM 동작을 제어하는 특수 속성 (dr-*)

Expression Binding

\${}$ and #{}# syntax for data binding in templates

템플릿에서 데이터 바인딩을 위한 \${}$와 #{}# 구문

Form Validation

Built-in validators for form data validation

폼 데이터 검증을 위한 내장 검증기
